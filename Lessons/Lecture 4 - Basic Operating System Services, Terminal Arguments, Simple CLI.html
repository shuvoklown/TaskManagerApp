Lecture 4.0: Basic Terminology

File System: The way an operating system keeps track of files and directories on a disk or partition.

Absolute Pathing: Absolute pathing is a term that is used to indicate the absolute path of a folder or file in a file system. For instance, C:\users\broge2\Documents\recipes.txt is an absolute path to the file recipes.txt on a Windows machine. On linux, it would look like this: /home/btcr/Documents/recipes.txt. Absolute pathing will always contain the root of the file system which in Windows is a drive letter such as _C:_ from the example above (C:\ is also where Windows always installs itself), and in Linux the root directory is a single forward slash, /, known as root.

Relative Pathing: Relative pathing is how we can navigate from one location to another without using absolute pathing. For instance, if you are in a directory called Projects and inside of this directory you have another directory, Python, you can just type cd Python. You don't have to give the absolute path to the Python directory. The path is relative to where you are currently at in the file system. You can also use a single period, ., to indicate the current directory so cd ./Python would indicate "in this directory move to the Python directory". You can equally navigate to the parent directory using two periods, ... If you are in the Python directory and want to move back to the Project directory you do cd ... You can string together directories using relative paths to navigate in the file system. Let's say you have another directory in Projects folder, Java, and you are located in Projects/Python/Calculator directory. You can cd ../../Java to move to the Java directory in Projects. This works if we look to see where we are at which is ./Projects/Python/Calc. The first .. moves us to /Projects/Python. The next .. moves us to /Projects. The /Java then moves us to the Java directory in the /Projects folder or /Projects/Java. This is the power of relative pathing.

When should you use relative over absolute pathing in your programs? That depends. If you have to access a file or folder that is part of the operating system like AppDir in Windows or /usr/share in Linux you should use absolute pathing. If you have to access a resource folder created by your program that is in the same directory that the script is ran in then you should use relative pathing.

Folder/Directory: Folders and directories are interchangeable names for a special file that contains a list of the location of other folders and files (technical definition). This is abstracted away by user space and you can just think of folders and directories as a container for other folders and directories.

All you need to run other people's Python scripts is the Python interpreter. As long as you have the Python interpreter installed you can run a .py script file. The base name is the last bit of of a path. Given C:\Windows\System32, the base name is System32.

Lecture 4.2: Command Line Parsing

If you've ever used the command line to perform tasks like grep, ssh, curl, netstat, nslookup, and many many many other operations one might have done as a super user then you know that these programs don't print questions to the user to manipulate how the program behaves. If that were the case, they would have to ask questions about every possible operation these commands can do which is not fast for people that just want to get something done. Instead, you can pass arguments to the program on the call line of the terminal you are using, press ENTER, and watch it go to work. We can do the same thing in Python using various modules.

For the examples in this section, it is best you write and run from a script file.

Lecture 4.2.1: sys.argv

sys is the simplest module that has a variable, argv, that contains a list of arguments passed to the program. Here is a very simple example.

from sys import argv

print(argv)

Now, call this with some arguments separated by spaces.

python scriptfile.py a b c d e

You should get something like this

['.\\scriptfile.py', 'a', 'b', 'c', 'd', 'e']

If you are wondering how you can pass a single argument that may contain spaces, you can surround the entire argument with double quotes.

Notice, the first item is the name of the script file. This is useful for us because we can use that in messages to the user and since they can rename the script file we won't have to bother doing checks or being vague. Here is an expansion that requires the user to pass at least 2 arguments.

from sys import argv

PROG_NAME = argv[0]

ARGUMENTS = argv[1:]

if len(ARGUMENTS) < 2:
    print(f"Usage: {PROG_NAME} arg1 arg2")
else:
    print(ARGUMENTS)

Call this the same way you normally run a Python file and only pass it 1 or no arguments. You should get a usage message that says

Usage: .\scriptfile.py arg1 arg2

Try changing the name of the scriptfile to something else and make the same call. You should see the usage message use the name of the new script file name.

Now call the script file with the right number of arguments. Suppose you pass it this.

python .\scriptfile.py This is a cool "Script File"

Instead, you should now get

['This', 'is', 'a', 'cool', 'Script File'] #Notice the "Script File" is one element instead of two

Now, let's write a script called makefiles.py. This script file will take a list of filenames and create them. This is kind of like touch in Linux that simply creates an empty file only we will make several empty files.

from sys import argv

PROG_NAME = argv[0]

ARGUMENTS = argv[1:]

if len(ARGUMENTS) < 1:
    print(f"Usage: {PROG_NAME} file1...fileN")
else:
    for filename in ARGUMENTS:
        print('Creating', filename)
        with open(filename, 'w'):
            pass

When you run this with 1 or more arguments, you will create files with the name of the passed arguments. Now suppose you want to expand this to make directories. You need to allow the user to indicated whether they want a directory or file. The concept of changing how the program works is usually called a switch or flag. Windows convention uses a / to indicate a switch on the command line, however, we will use the Linux convention as this is also the convention used for the Python modules later. Linux uses a hyphen, commonly stated as tack or dash, to indicate a switch. Some switches accept other arguments while others are just indicators. For this example, we will simply use a switch as an indicator of whether to create files or folders.

-f for files and -d for directories.

from sys import argv
import os

def make_files(filenames):
    for filename in filenames:
        print('Creating file', filename)
        with open(filename, 'w'):
            pass

def make_directories(directories):
    for directory in directories:
        print("Creating directory", directory)
        os.mkdir(directory)

PROG_NAME = argv[0]

ARGUMENTS = argv[1:]

if len(ARGUMENTS) < 1:
    print(f"Usage: {PROG_NAME} arg1 arg2")
elif ARGUMENTS[0] == '-f':
    make_files(ARGUMENTS[1:])
elif ARGUMENTS[0] == '-d':
    make_directories(ARGUMENTS[1:])
else:
    print(f"Usage: {PROG_NAME} -[f or d] name...nameN")

Now our script is getting more complicated but also a little restrictive. For instance, the switch -f and -d must be the first thing passed to the program. Switches can sometimes be placed anywhere on the command line when passed to a program. Also, with this code we can only create folders and files instead of say mixing the switches to allow both for a single call. We most certainly can add logic to do this and much more, however, the Python developers have us covered. We can use the much more powerful argparse module.

Lecture 4.2.2: argparse

The argparse module allows us to construct a set of rules for how to parse incoming arguments to our program. There is a lot that you can do so I will cover the basics of creating and using argparse and some of the options you can use to create a command line program.

We start by creating an ArgumentParser object. The constructor for the ArgumentParser object can take many arguments.

Python 3 Documentation: ArgumentParser Constructor

import argparse

parser = argparse.ArgumentParser(description="A program that accepts arguments via the command line")

parser.print_help()

The call to print_help will be the same as if we passed -h or --help to our program but we have not actually parsed anything yet. When you run this program you should get a usage line, the description as stated in the program, and the list of arguments. The only arguments right now are optional arguments -h and --help but again, these do nothing. (Remove the call to print_help and pass -h or --help to see what I mean).

Once you have an ArgumentParser object you need to start adding switches and how the behave such as how they store data, how much data should they store if they even do store datay, are the switches required, and many other options. We can also specify regular arguments. Remember, switches are designated by being preceded with a hyphen or two. Before we begin, let me go over some terminology.

Switch: This is a type of argument to our program that dictates how our program behaves. A switch is designated by one or two hyphens.

Short Name: A short name for a switch is a single letter and is preceded by a single hyphen. For instance, -h is the shortname.

Long Name: A long name for a switch is preceded by two hyphens and is usually a word or multiple names. For instance, --help is the long name.

Argument: This refers to items that are both switches and regular items. For instance, in our previous program that takes a list of names and creates files or folders, -f or -d are switches and the names of the files and folders are just for the program to create those. There are subtypes of arguments as well.

Positional Argument:  This is a type of argument that in the command line has meaning depending on where it is placed.  These typically are not flags or switches but switches can come before or after a positional argument for processing.

Optional Arguments:  These are not necessary and are usually only used to change the behavior of the program.

Required Arguments:  These arguments are required by your program in order to run.  Positional arguments are by default required.

After you are done adding arguments, you need to parse the arguments and a Namespace object is returned. Your switches then become instance variables of the Namespace object that we can use like normal variables that store information. Let's try a simple example.

import argparse

parser = argparse.ArgumentParser(description="A program that accepts arguments via the command line")

parser.add_argument('firstname')

args = parser.parse_args()

print(args)
print("firstname is", args.firstname)

Call your script file with your first name and see what is returned. Also, try passing -h and --help. You will see that firstname is a positional argument. Let's expand this to accept our last name but is optional by using a switch -l or --lastname.

import argparse

parser = argparse.ArgumentParser(description="A program that accepts arguments via the command line")

parser.add_argument('firstname')
parser.add_argument('-l', '--lastname')

args = parser.parse_args()
full_name = args.firstname

if args.lastname:
    full_name += ' ' + args.lastname

print(args)
print("Full name is", full_name)

Now when you call this program, you can pass your last name using -l. For example

python namer.py -l Rogers Brian

OR

python namer.py Brian -l Rogers

Notice, that the switch -l can come before or after the positional argument.

There is a lot we can do with the add_argument function. I'm only going to cover a few more options but please to go over the documentation just so you know what you can do.

Python 3 Documentation: add_argument

import argparse

parser = argparse.ArgumentParser(description="A program that accepts arguments via the command line")

parser.add_argument('firstname')
parser.add_argument('-l', '--lastname')
parser.add_argument('-a', '--age', type=int, default=None)
parser.add_argument('-n', '--nicknames', nargs='+')

args = parser.parse_args()

full_name = args.firstname

if args.lastname:
    full_name += ' ' + args.lastname

print(args)
print("Full name is", full_name)

if args.age:
    print(f"You are {args.age} years old and will be {args.age + 10} in 10 years")
if args.nicknames:
    print(f"Your nicknames are {', '.join(args.nicknames)}")

Using the type option we can change the type expected by the parser. By default, types of all parameters are str or string. Using type=int we should expect a value of int. This will throw an error otherwise.

Using default, we can say what the default value should be. In this case, I explicitly state None since the default of an int could be 0. If the argument is not used then it will be set to None in the Namespace object.

Using nargs, I can specify how many arguments after the switch I am expecting. In this case, if you use the -n or --nicknames on the command line I expect 1 or more items. This will be stored as a list by the Namespace object.

ArgParse is a powerful CLI (Command Line Interface) library to allow us to create command line programs and focus on the actual program instead of worrying about the parsing logic. If you are doing something simple, this may be overkill but if you want to expand what your command line program does then this is definitely the goto.

Lecture 4.3: glob

This section of the lecture should go with the files and folders but I wanted to introduce argument parsing so you can attempt the try out problems.

The glob module allows us to specify a pattern to match a list of files and folders. Suppose you have a large project that uses many different languages and files and you only want the .py or .json files or have directories with -debug in their name to indicate that they are only debug modules that should not be sent with production code. Instead of having to list all of the files and folders and going through by hand you can use pattern matching.

Python 3 Documentation: glob

When pattern matching we use the following

*: The asterisks is a wildcard that matches 0 or more characters.

?: The question mark is also a wildcard but matches 0 or 1 characters.

Ranges: Ranges can be specified using square brackets. For instance, [a-h] matches all lowercase characters from a to h or manually, [ostz] which skips some letters.

import glob

for g in glob.glob('*/*.py', recursive=True):
    print(g)

First, I specified explictly to be recursive which means it should traverse directories and subdirectories that match the pattern, in this the wildcard *.

Second, this will look for any and all .py files and print them.

Since glob does pattern matching on files and folders then I needed to explicitly specify any and all directories in order to be recursive. Remove the */ and notice that it may not traverse directories and subdirectories even though recursive is set to True.

If we wanted to traverse other directories, regardless of pattern, use os.walk and os.path.

import glob
import os

for root, dirs, _ in os.walk('.'):
    for directory in dirs:
        path = os.path.abspath(os.path.join(root, directory))
        for g in glob.glob(os.path.join(path, '*.py')):
            print(g)

Lecture 4.4: Time

Time is a weird construct that is actually hard to get right. I recommend this Computerphile video.

Computerphile: The problem with Time and Timezones

That's why I wanted to talk about it here. The different calls will return different results. The one you need to watch out for is clock as it doesn't return what you think it returns.

time() returns the number of seconds since the epoch.
ctime() returns a human readable version of time.
clock() returns processor clock time. Returns seconds.
sleep() takes seconds and pauses the program for that many seconds. Seconds is a float so you can pause for half seconds or .333 seconds, etc.

import time

print("Epoch", time.time())
print("Readable", time.ctime())
print("Processor", time.clock())
time.sleep(5)
print("Prints after 5 seconds")

Epoch 1528333359.971539
Readable Wed Jun  6 20:02:39 2018
Processor 2.4380925631593975e-07
Prints after 5 seconds

To demonstrate that time.sleep works (and to have a little fun) we can do a little animation. In a script file, write the following.

import os

CLEAR_TERM = None

Lecture 4.5: Third Party Modules

Everything you have seen so far is located in the standard library. Now we will talk about how we can install third party modules that others wrote not installed here. It is simple as we just need to use the pip command. First, to ensure you have pip, run the following.

WARNING Do not overwrite the default pip used by your operating system. If you are on Linux or Mac then any pip command will be pip3. For the below, be sure to use python3 -m ensurepip if python3 is not your default.

python -m ensurepip --default-pip

Now say we want to install the requests module. You simply need to run

pip install requests

Voila! Simple. Other useful commands are

pip list will list all installed modules.
pip install --upgrade [package] will upgrade any packages.
python -m pip install --upgrade pip will upgrade pip.

This installs packages from PyPi by default, however, if you give a URL instead of a package, say from github, then it will install from there.

Finally, if the package doesn't reside on github or PyPi or from any other place that you can easily have pip handle then make sure to read the README of the project. If it does not contain a README then chances are it's not a complete or full release package yet.
