Lecture Organization

The lectures are separated into different sections. These sections are described below.

Lecture Sections

Each lecture section will contain the topic that portion of the lecture covers, a try out section and a problem section which are discussed below. At the end of each lecture section may be a reference link to a documentation page. It is recommended to at least skim the page so you know what it contains. The documentation will be a huge part of learning Python since it is a community driven language.

Try Out Sections

These sections give you the opportunity to experiment with what you just learned. Typically, these sections contain a snippet of code that is used to demonstrate a concept, give you some practice, or to show given behavior for certain syntax and usually it's all three. Take the time during the try out portions of the lectures to change the code and see what happens inside the Python terminal.

Exercise Sections

These sections give you a chance to solve some problems with the new syntax you just learned. If you have problems solving these reread the problem and make sure you understand what you are being asked. If you are still struggling then go back through the lecture and reread any section of the lecture that didn't make sense to you. If you are still struggling then you can check the end of the lecture for a possible solution. Try to follow the solution on paper.

Code Sections

There are many code sections which show Python code either via a command prompt or in a script file. If the code is in a script file, the name of the file will be at the top as a comment such as # scriptname.py. This is to indicate to you that you should write this code in a .py file (You don't have to name it the same as what is shown) and run it using the command python scriptname.py.

The other code sections will be a command prompt. These will not have a script name at the top. Since I am writing these lectures using Jupyter Notebook so I can convert them to different formats, you will see IN[#] for the code you should write in a prompt and OUT[#] for what result you should see. In your prompt you won't see IN or OUT but >>>.

Fixing Errors Sections

Some lectures may contain a Fixing Errors Section. This is where a list of the common problems that may occur while working through the lecture will appear, an explanation of why the error occurred, what to watch out for, and how to fix it. Use this section to help you understand when problems arise so you can learn how to fix them.

Installation

Notification for Linux and Mac Users

Linux and Mac OS already have Python installed. To check which version, you should open a terminal and type python --version and press Enter. You will see the version of Python installed. If you do not see a 3.x.y, where x and y are other numbers, then you need to install 3.x.y.

If you have Python installed but it is not Python 3 then you need to install Python 3. See Picking a Version below.

They both can coexist on the same machine, however, if you install 3 where 2 was the default then you will need to make sure to run python3 at the command line to run both a script file and to start the interpreter shell.

Picking a version

We are using Python 3 for this course. The reason being Python 2 is losing support in 2020 and there is no reason to teach you outdated information. There was a large community that maintained Python 2 for the longest time but 3 has come so far that the new stuff added actually makes our lives easier as Python developers. Python 2 will not be accepted for this course anymore. More information on the loss of support for Python 2: Python 2 Clock Countdown.

Fixing Errors

Windows Installation

If you installed Python but get the following error below

'python' is not recognized as an internal or external command,
operable program or batch file.

then you may not have python.exe in your PATH environment.

  1. To get python.exe in your PATH navigate to where you have Python installed. The default location is in the C:\PythonXY where XY is your version. Copy the folder location.
  2. Now, open start and right-click Computer and select Properties.
  3. On the left hand side bar, click on Advance System Settings. Make sure you are on the Advanced tab.
  4. Click on the Environment Variables... button.
  5. Under system variables, scroll down until you see Path. Select it by left-clicking it.
  6. Click Edit. You will be presented with a window that contains a Variable Name and Variable Value. It is important you do not change the variable name nor delete the variable value.
  7. Click inside the Variable Value text box and press the End key. This will take you to the end of the list.
  8. At the cursor location, type a semicolon and the path to the Python folder.
    • Example: C:\Program Files\Calibre2\; C:\Python36\
      The example shows what the Variable Value looks like at the end if you had, say, Calibre installed. The folder that contains the python.exe is pasted after a semicolon.
  9. Press OK then press OK again.

Validate this worked by opening up the command prompt again, closing it if it was already open, and type python and press Enter. You should see the Python header and the >>> prompt. If you do not, then you may need to reinstall Python.

Script vs Interactive Shell

There are two modes of running Python and they are via a script file or in the interactive shell (also known as a REPL, read-eval-print-loop). Both have their pros and cons and knowing when to use one over the other will make you a more productive programmer (and a more knowledgeable one at that).

Interactive Shell

The interactive shell can be started in either the Command Prompt (Windows) or Terminal (Linux/Unix) by typing python and nothing after it. (We will see the other options that can accompany it). This should start a REPL and wait for input with the prompt >>>. From here, you can start to write snippets of python code. This is great as you can easily test something you want to try like an algorithm, a new method, or to make sure you understand how something works. Another benefit is that you don't need to use the print() function to get information from a variable, function, or method. Try the following below. (Normally, you would see ... to indicate something has been removed for clarity, however, ... has meaning in the interactive shell so I decided to use three tildes, ~~~)

>>> 100 * 200
~~~ #Do not write this.  This should be the output of the above math.
>>> a = _ / 10 # Take the output of the last call, the underscore, and divide by 100
>>> a
~~~ #Do not write this.  This should be the output of the value 
~~~ #contained in variable a

This is great if you are just testing something, however, if you are writing a bigger project, if a bug occurs or there is a syntax error, you would have to rewrite all of that portion of broken code. Try this. (For the line under the function, make sure to include 4 spaces in front. We'll get more into Python blocks in later lecures.)

>>> def func(a):
...     print(a + " * 2 = " + (a * 2))
>>> func(4)
~~~ #Should receive an error

Now, if we tried to fix this, we would have to rewrite the entire function starting with def func(a)...etc. Imagine trying to do this with classes or libraries. That's why we will only use the shell to demonstrate fundamentals and use script files for large projects.

The Python Command

If you've never used a command prompt or terminal before, you'll want to get acquainted with it. Even if you are using an IDE, when reading how to do something online, you might come across options you can pass to the python command called flags or switches that may or may not contain arguments. A good example is the --help flag. In your command prompt/terminal type python --help and press enter. Here, you will get a list of the flags and whether they require an argument and what they do. A flag is preceded with either one or two hyphens (like in the help flag earlier). An argument is just a name after the flag like the cmd argument that is part of the -c flag. This means that when we use -c we have to pass it some argument (in this case, a Python expression). Try the following

python -c "print('a' * 100)"

This will run the print('a' * 100) as actual Python code, not in an interactive shell nor in a script file, in the command prompt/terminal. Explore, or Google, the Python command line environment and try out some of the flags, if you are able.

For more information visit Command Line and Environment

Script Files

To write a script file it is as easy as opening up a text editor, writing some code, and running it in the terminal. If you use an IDE, then you don't even need to open a terminal to run it as the IDE should take care of that for you. Just remember to give your files the .py extension. Let me give an example and go from there.

Create a projects folder somewhere on your system. This is where all your projects will get stored. If you are using an IDE this is typically called your workspace. Inside that project folder, create a script_test folder. Inside that folder create a test.py file. Open that file in an editor of your choosing and write the following.

print("This is easy")
print("Python is fun")
print("My name is <insert your name here>")

Now, open up a terminal and navigate to your directory. On all platforms, you will use the cd command to navigate your directory structure. Use ls in linux or dir in windows to get a list of directories in the location of where you are located. Here is an example using Windows and assuming the file is in C:\Users\broge2\Documents\Python\script_test:

C:\Users\broge2>dir
~~~
04/18/2017  07:13 PM    <DIR>          Documents
~~~
C:\Users\broge2>cd Documents
C:\Users\broge2\Documents>dir
~~~
11/16/2016  05:13 PM    <DIR>          Python
~~~
C:\Users\broge2\Documents>cd Python
C:\Users\broge2\Documents\Python>dir
~~~
07/04/2016  02:13 AM    <DIR>          script_test
~~~
C:\Users\broge2\Documents\Python>cd script_test
C:\Users\broge2\Documents\Python\script_test>dir
~~~
04/27/2017  04:50 PM                71 test.py
~~~

Make sure to use your folder names when navigating the file system. Now you can run the script file. Type python test.py and press Enter. May need to issue python3 instead. You should see the output of the print functions.

Writing script files will become more useful when we start writing bigger programs. By writing a script file, you can edit it witout need of rewriting all of the other code to make it run. It is saved so you can come back to it later if you have to. It allows you to organize your code better.

Really, the only thing that I would say is a con, is you MUST use print() to write to standard out so if you want to see the result of a calculation or algorithm you have to print() it.
